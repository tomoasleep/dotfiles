# Playwright Test Dir Setup

あなたのタスクは、ローカル動作検証用に `.playwright-tests` ディレクトリの作成、及び初期動作環境のセットアップを行うことです。
このディレクトリは、アプリケーション本体やそれのテストとは独立して、都度の動作確認を行うための役割を持ちます。

## 技術的要件

- アプリケーション本体とは独立した npm package として管理します。
  - パッケージマネージャーは `pnpm` を使用してください。
- `.playwright-tests/` ディレクトリ内で `pnpm exec playwright test` でテストが実行できる状態にします。
- アプリケーションのサーバーを起動するためのタスクを `dev-up` script として作成します (`pnpm dev-up` で起動できるようにしてください)
- テストフレームワークは Playwright を使用します。

## やること

`.playwright-tests/` ディレクトリ内で、技術的要件を満たすためのテスト環境と、サーバーが起動していることを確かめる簡単な playwright test が作られている状態にすることです。

なお、完了前に必ず動作確認し、テストが行える状態を持って終了としてください。

## 禁止事項

- `.playwright-tests` 外のファイルを変更してはいけません。動作確認方法が調査して不明でテストが行えない場合は、作業を中断し、ユーザーに相談してください。
- 動作していることを実際に確認せずに完了したと報告してはいけません。

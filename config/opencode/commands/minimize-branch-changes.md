あなたのタスクは、現在のブランチの変更をブランチや計画に最低限必要な変更に縮小させることです。

チーム開発のプラクティスとして、

- Pull Request は小さくすべき
- Pull Request での変更は、1つの目的に集中させる

という方針があるので、目的にそぐわない変更、目的には必要ない変更を取り除きます。

## TODO

1. 現在のブランチの目的を把握
  - 以下の調査を general agent に依頼
    - 紐づいている Pull Request の内容、関連ドキュメントの内容の確認 
  - 目的として不明な点があれば question tool を使ってユーザーに確認 (※ Plan mode でも利用可能)
2. 現在のブランチの変更内容のうち、必要ない変更の候補を explore agent に委託して洗い出す
3. ユーザーに、洗い出した不要変更候補を提示し、削除 or 復元して良いか確認 (question tool を使用)
4. 実施
  - ユーザーからの承認が得られた変更をそれぞれ general agent に依頼して削除 or 復元
    - 指示は具体的に行い、それ以外の作業を行わないようにすること
